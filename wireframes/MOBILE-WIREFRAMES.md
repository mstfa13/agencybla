## âœ… MOBILE WIREFRAME PRINCIPLES APPLIED

### 1. **Touch Target Sizing**
- All buttons: Minimum 44Ã—44px (Apple) or 48Ã—48px (Material)
- Input fields: Minimum 48px height
- Links with icons: Minimum 44Ã—44px tap area

### 2. **Typography for Mobile**
- Body text: 16px minimum (prevents iOS auto-zoom)
- Line height: 1.6-1.8 for readability
- Headings: 28-32px (H1), 24px (H2), 18-20px (H3)
- Short paragraphs: Max 3-4 lines (60-80 characters)

### 3. **Spacing & Whitespace**
- Screen padding: 24px left/right (safe area)
- Section spacing: 40-60px vertical
- Component spacing: 16-24px between cards/elements
- Avoid cramped layouts (use generous whitespace)

### 4. **Navigation Patterns**
- Sticky header (56px) with hamburger menu
- Breadcrumbs (optional, collapsible)
- Back button (top-left on detail pages)
- Sticky bottom bar with CTAs (56px)

### 5. **Content Prioritization**
- Most important content above the fold (650px)
- Primary CTA visible within first scroll
- Progressive disclosure (accordions, tabs, "Show more")
- Eliminate unnecessary elements on mobile

### 6. **Performance Optimization**
- Hero images: < 100KB (WebP format)
- Lazy-load all below-fold images
- Inline critical CSS (< 15KB)
- Defer non-critical JavaScript
- Use system fonts for speed

### 7. **Form Optimization**
- Large input fields (48px height)
- Appropriate input types (email, tel, number)
- Auto-capitalize where needed
- Minimal required fields (3-5 max)
- Clear validation messages

### 8. **Visual Hierarchy**
- Use size, color, and spacing to guide eye
- High contrast for readability (WCAG AA minimum)
- Consistent icon system (48px or 32px)
- Clear section separation (borders, backgrounds, spacing)
